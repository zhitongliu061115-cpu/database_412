# 安全管理开发说明

## 1. 目标

本次仅完成 `安全管理` 模块，不包含 `数据库恢复` 模块。

依据需求文档，本模块实现：

- 用户管理
- 登录与退出
- 表级对象权限授予与收回
- SQL 执行前的权限检查

## 5. 模块设计

### 5.1 SecurityManager 职责

`SecurityManager` 负责：

- 初始化默认管理员
- 用户增删查
- 登录状态维护
- 权限授予与回收
- 执行权限判断
- 用户/权限持久化

### 5.2 默认管理员

程序启动时如果不存在用户文件，会自动创建默认管理员：

- 用户名：`admin`
- 密码：`admin123`

初始化位置：

- `database_412/main.cpp`

## 6. 持久化文件

本模块新增两个系统文件，保存在项目根目录：

- `users.db`
- `privileges.db`

### 6.1 users.db

存储用户信息：

- 用户名
- 密码摘要
- 是否管理员
- 创建时间

### 6.2 privileges.db

存储权限信息：

- 用户名
- 数据库名
- 表名
- 权限掩码
- 更新时间

## 7. 权限模型

当前实现的是表级对象权限。

支持权限：

- `SELECT`
- `INSERT`
- `UPDATE`
- `DELETE`
- `ALTER`
- `DROP`
- `ALL`

管理员特性：

- 管理员默认拥有所有权限
- 只有管理员可以执行用户管理和授权管理

## 8. SQL 语法

### 8.1 用户管理

```sql
CREATE USER alice IDENTIFIED BY '123456';
DROP USER alice;
LOGIN alice '123456';
LOGOUT;
```

### 8.2 授权管理

```sql
GRANT SELECT, INSERT ON student TO alice;
REVOKE INSERT ON student FROM alice;
GRANT ALL ON student TO alice;
```

### 8.3 查询管理信息

```sql
SHOW USERS;
SHOW GRANTS FOR alice;
```

## 9. 权限校验接入点

### 9.1 管理员限制

下列操作限制为管理员：

- `CREATE DATABASE`
- `DROP DATABASE`
- `CREATE TABLE`
- `CREATE USER`
- `DROP USER`
- `GRANT`
- `REVOKE`
- `SHOW USERS`

### 9.2 表级权限限制

下列操作会检查当前用户对当前数据库下目标表的权限：

- `SELECT` -> `PRIV_SELECT`
- `INSERT` -> `PRIV_INSERT`
- `UPDATE` -> `PRIV_UPDATE`
- `DELETE` -> `PRIV_DELETE`
- `ALTER TABLE` -> `PRIV_ALTER`
- `DROP TABLE` -> `PRIV_DROP`

## 10. 解析器改动

## 13. 建议测试流程

### 13.1 管理员初始化

启动服务端后，应出现默认管理员初始化提示。

### 13.2 基本用户流程

```sql
LOGIN admin 'admin123';
CREATE DATABASE school;
USE school;
CREATE TABLE student;
ALTER TABLE student ADD id INT NOT NULL;
ALTER TABLE student ADD name VARCHAR(20);
CREATE USER alice IDENTIFIED BY '123456';
GRANT SELECT, INSERT ON student TO alice;
SHOW USERS;
SHOW GRANTS FOR alice;
LOGOUT;
LOGIN alice '123456';
USE school;
INSERT INTO student VALUES (1, 'Tom');
SELECT * FROM student;
DELETE FROM student WHERE id = 1;
```

预期：

- `INSERT` 成功
- `SELECT` 成功
- `DELETE` 因无权限失败

### 13.3 回收权限

```sql
LOGOUT;
LOGIN admin 'admin123';
USE school;
REVOKE INSERT ON student FROM alice;
LOGOUT;
LOGIN alice '123456';
USE school;
INSERT INTO student VALUES (2, 'Jerry');
```

预期：

- `INSERT` 因权限被收回而失败
