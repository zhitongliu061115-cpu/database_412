# 安全管理完整测试流程

## 1. 目的

本文档用于从零开始完整测试当前项目的“安全管理”模块。

测试目标：

- 验证默认管理员初始化
- 验证用户创建与删除
- 验证登录与退出
- 验证授权与回收
- 验证业务语句的权限拦截

## 2. 测试前准备

### 2.1 关闭后端程序

确保 `database_412.exe` 没有在运行。

### 2.2 清理旧测试数据

如果想从完全干净的状态开始，删除下面这些文件和目录：

- `users.db`
- `privileges.db`
- `system.db`
- `data/`

说明：

- 这样下次启动时会重新初始化默认管理员
- 如果你不想删除原有业务数据，可以只删 `users.db` 和 `privileges.db`

### 2.3 编译后端

打开 Visual Studio，编译：

- 解决方案：`database_412.sln`
- 工程：`database_412`
- 配置：`Debug | x64`

生成后可执行文件位置：

- `x64/Debug/database_412.exe`

## 3. 启动系统

### 3.1 启动后端

运行：

- `x64/Debug/database_412.exe`

预期：

- 服务端启动
- 控制台显示监听 `8080`
- 第一次启动时会初始化默认管理员

默认管理员：

- 用户名：`admin`
- 密码：`admin123`

### 3.2 启动客户端

运行 Qt 客户端工程：

- `DBMS_Client`

在客户端中：

1. 点击“连接服务端”
2. 等待连接成功
3. 在 SQL 输入框中按下面步骤执行

## 4. 第一阶段：默认管理员登录

执行：

```sql
LOGIN admin 'admin123';
```

预期：

- 登录成功
- 日志中出现管理员登录成功提示

如果失败：

- 检查 `users.db` 是否被误删后未重新启动服务端
- 检查密码是否仍为 `admin123`

## 5. 第二阶段：管理员创建测试库和测试表

### 5.1 创建数据库

执行：

```sql
CREATE DATABASE secdb;
```

预期：

- 数据库创建成功

### 5.2 切换数据库

执行：

```sql
USE secdb;
```

预期：

- 切换数据库成功

### 5.3 创建表

执行：

```sql
CREATE TABLE student;
```

预期：

- 表创建成功

### 5.4 添加字段

执行：

```sql
ALTER TABLE student ADD id INT NOT NULL;
ALTER TABLE student ADD name VARCHAR(20);
```

预期：

- 两条语句都成功

## 6. 第三阶段：管理员创建普通用户

### 6.1 创建用户 alice

执行：

```sql
CREATE USER alice IDENTIFIED BY '123456';
```

预期：

- 用户创建成功

### 6.2 查看用户列表

执行：

```sql
SHOW USERS;
```

预期：

- 至少能看到：
  - `admin`
  - `alice`

## 7. 第四阶段：管理员给 alice 授权

### 7.1 授予查询和插入权限

执行：

```sql
GRANT SELECT, INSERT ON student TO alice;
```

预期：

- 授权成功

### 7.2 查看授权结果

执行：

```sql
SHOW GRANTS FOR alice;
```

预期：

- 能看到 `student` 表相关授权
- 权限应至少包含：
  - `SELECT`
  - `INSERT`

## 8. 第五阶段：切换为普通用户 alice

### 8.1 管理员退出

执行：

```sql
LOGOUT;
```

预期：

- 退出成功

### 8.2 alice 登录

执行：

```sql
LOGIN alice '123456';
```

预期：

- 登录成功

### 8.3 切换数据库

执行：

```sql
USE secdb;
```

预期：

- 切换成功

## 9. 第六阶段：验证 alice 的允许操作

### 9.1 插入数据

执行：

```sql
INSERT INTO student VALUES (1, 'Tom');
INSERT INTO student VALUES (2, 'Jerry');
```

预期：

- 都成功

### 9.2 查询数据

执行：

```sql
SELECT * FROM student;
SELECT id, name FROM student;
SELECT * FROM student WHERE id = 1;
```

预期：

- 都成功
- 能查询到刚插入的数据

## 10. 第七阶段：验证 alice 的越权操作

alice 当前只有 `SELECT` 和 `INSERT`，没有 `DELETE`、`UPDATE`、`ALTER`、`DROP`。

### 10.1 测试 DELETE

执行：

```sql
DELETE FROM student WHERE id = 1;
```

预期：

- 失败
- 提示缺少 `DELETE` 权限

### 10.2 测试 UPDATE

执行：

```sql
UPDATE student SET name = 'Jack' WHERE id = 1;
```

预期：

- 失败
- 提示缺少 `UPDATE` 权限

### 10.3 测试 ALTER TABLE

执行：

```sql
ALTER TABLE student ADD age INT;
```

预期：

- 失败
- 提示缺少 `ALTER` 权限

### 10.4 测试 DROP TABLE

执行：

```sql
DROP TABLE student;
```

预期：

- 失败
- 提示缺少 `DROP` 权限

## 11. 第八阶段：管理员回收 alice 的 INSERT 权限

### 11.1 alice 退出

执行：

```sql
LOGOUT;
```

### 11.2 管理员重新登录

执行：

```sql
LOGIN admin 'admin123';
USE secdb;
```

### 11.3 回收 INSERT 权限

执行：

```sql
REVOKE INSERT ON student FROM alice;
```

预期：

- 回收成功

### 11.4 查看 alice 当前授权

执行：

```sql
SHOW GRANTS FOR alice;
```

预期：

- `INSERT` 已不在权限列表中
- `SELECT` 仍保留

## 12. 第九阶段：验证回收后的效果

### 12.1 管理员退出

执行：

```sql
LOGOUT;
```

### 12.2 alice 重新登录

执行：

```sql
LOGIN alice '123456';
USE secdb;
```

### 12.3 再次测试 INSERT

执行：

```sql
INSERT INTO student VALUES (3, 'Alice');
```

预期：

- 失败
- 提示缺少 `INSERT` 权限

### 12.4 再次测试 SELECT

执行：

```sql
SELECT * FROM student;
```

预期：

- 仍然成功

## 13. 第十阶段：测试删除用户

### 13.1 alice 退出

执行：

```sql
LOGOUT;
```

### 13.2 管理员登录

执行：

```sql
LOGIN admin 'admin123';
```

### 13.3 删除 alice

执行：

```sql
DROP USER alice;
```

预期：

- 删除成功

### 13.4 查看用户列表

执行：

```sql
SHOW USERS;
```

预期：

- 不再显示 `alice`

### 13.5 验证 alice 无法登录

执行：

```sql
LOGOUT;
LOGIN alice '123456';
```

预期：

- 登录失败
- 提示用户不存在或登录失败

## 14. 整体验收结论

如果下面这些都满足，说明安全管理模块通过基本测试：

1. 默认管理员能自动初始化
2. 管理员能创建和删除用户
3. 用户能登录和退出
4. 管理员能授权和回收权限
5. 普通用户只能执行被授权的表级操作
6. 越权操作会被拦截
7. 权限回收后立刻生效
8. 用户删除后无法再次登录

## 15. 推荐最终演示脚本

如果老师只看一遍演示，建议按下面顺序：

```sql
LOGIN admin 'admin123';
CREATE DATABASE secdb;
USE secdb;
CREATE TABLE student;
ALTER TABLE student ADD id INT NOT NULL;
ALTER TABLE student ADD name VARCHAR(20);
CREATE USER alice IDENTIFIED BY '123456';
GRANT SELECT, INSERT ON student TO alice;
SHOW GRANTS FOR alice;
LOGOUT;

LOGIN alice '123456';
USE secdb;
INSERT INTO student VALUES (1, 'Tom');
SELECT * FROM student;
DELETE FROM student WHERE id = 1;
LOGOUT;

LOGIN admin 'admin123';
USE secdb;
REVOKE INSERT ON student FROM alice;
LOGOUT;

LOGIN alice '123456';
USE secdb;
INSERT INTO student VALUES (2, 'Jerry');
SELECT * FROM student;
```

预期演示点：

- 授权前后行为不同
- `SELECT` 能执行
- `INSERT` 先成功后失败
- `DELETE` 一直失败
